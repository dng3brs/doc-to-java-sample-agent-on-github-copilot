# Remotionによる亀の魅力解説動画 制作要件定義および手順書

本文書は、Remotionを使用した動画制作における要件定義、技術選定、および制作手順について合意・決定した内容をとりまとめたものです。

---

## 1. 動画基本要件

### 1.1 テーマ・目的

- **テーマ**: 亀の魅力を伝え、飼うメリットおよび導入（飼育スタート）ガイドを紹介する。
- **トーン＆マナー**: 過度な演出を控え、実直に魅力をアピールする。シンプルでありながら伝わりやすく、センスを感じさせるデザインを目指す。

### 1.2 動画構成（アジェンダ）

全体の長さは1分〜1分半程度を想定し、以下の4セクション構成とします。

1. **イントロダクション**: オープニングと亀の基本的な魅力の提示
2. **亀を飼うメリット**: 静音性、省スペース、手間の少なさなどの訴求
3. **飼育スタートガイド**: 必要な初期セットや心構えなどの導入紹介
4. **アウトロ**: 全体のまとめと締めくくり

### 1.3 動画仕様

- **画面サイズ・解像度**: 16:9（1920 × 1080 / フルHD）
- **フレームレート**: 30 fps

## 2. ビジュアル＆音響デザイン方針

### 2.1 カラーパレット

- **メインカラー**: アースグリーン（自然や亀を連想させる落ち着いた緑系）
- **ベースカラー**: ライトグレー／オフホワイト（清潔感があり、視認性を高める明るい色調）
- **アクセントカラー**: ウォームサンド／ベージュ（温かみや柔らかさを補足）

### 2.2 タイポグラフィ（フォント）

- **採用フォント**: Noto Sans JP（Google Fonts）
- **スタイル**: ゴシック体（ウェイトは Medium / Bold を中心に使用し、視認性を担保）

### 2.3 素材利用方針

- **実写素材（写真・動画）**: フリー素材サイト（Unsplash, Pexels等）の高画質な実写素材を使用し、Remotionの `<Img/>` や `<OffthreadVideo/>` コンポーネントで読み込む。
- **図解・アイコン**: 最小限のシンプルなSVG線画アイコンを使用し、実写素材を引き立てる。

### 2.4 音響構成

- **BGM**: アコースティックギターやピアノを中心とした、主張しすぎない静かなロイヤリティフリー音源を使用。
- **ナレーション**: テキストテロップを主体とし、必要に応じてRemotionの `<Audio/>` タグを用いて合成音声（Voicepeakや音読さん等）の音声を再生。
- **効果音（SE）**: 場面転換時などに限定し、最小限に抑える。

## 3. 技術仕様・コード設計方針

### 3.1 技術スタック

- **プログラミング言語**: TypeScript（型定義による堅牢な実装）
- **CSSスタイリング**: Tailwind CSS（直感的なレイアウト調整とデザイン維持）

### 3.2 コンポーネント構造・データ管理

シナリオデータとUIロジックを分離し、メンテナンス性を向上させます。

- **データ定義 (`config/data.ts`)**: 各セクションのテキスト、使用画像パス、表示フレーム数（duration）等を定義。
- **ルート定義 (`src/Root.tsx`)**: `<Composition>` の登録・基本設定。
- **メインシークエンス (`src/MainSequence.tsx`)**: Remotionの `<Series>` または `<Sequence>` を用いて各セクションを順次再生。
- **セクションコンポーネント (`src/sections/`)**: イントロ、メリット、導入ガイド、アウトロの画面描画を担当。

### 3.3 アニメーション実装方針

外部の過剰なエフェクトライブラリは使用せず、Remotion組み込みの関数を用いて静かで滑らかなアニメーションを実現します。

- **アニメーションロジック**: `interpolate` および `spring`（`config: { damping: 200 }` 等で減衰を利かせた滑らかな動き）を使用。
- **演出内容**:
  - テキストや画像のフェードイン／フェードアウト（`opacity` 制御）
  - わずかなスライドイン（`translateY` による下からの移動）
  - 場面転換はシンプルなクロスディゾルブや余白を生かした切り替え

## 4. 制作ワークフロー（手順）

以下の5ステップで開発・制作を進めます。

1. **プロジェクト初期化 & 環境構築**
   - Remotion（TypeScript + Tailwind CSS）プロジェクトのセットアップ
   - フォント（Noto Sans JP）の読み込みおよび静的アセット（画像・音源）の配置
2. **データ構造・構成の定義**
   - `data.ts` に各セクションのテキスト、画像パス、表示時間（フレーム数）を定義
3. **コンポーネント実装**
   - 各セクションの画面レイアウトおよびコンポーネントを作成
   - `interpolate` / `spring` を使用したアニメーションおよび音声（BGM・SE）の組み込み
4. **プレビュー & 微調整**
   - `npx remotion preview` を実行し、ブラウザ上でタイミングやレイアウトを微調整
5. **レンダリング（書き出し）**
   - `npx remotion render` を実行し、最終的なMP4動画ファイルを生成
